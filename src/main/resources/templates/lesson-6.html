Pelajaran 6: Alpine.js Basics

Menambahkan reactivity di client-side dengan Alpine.js untuk interaksi yang lebih dinamis

ü§î Kenapa Perlu Alpine.js?

Pertanyaan: Pelajaran sebelumnya sudah canggih dengan HTMX saja, kenapa harus tambah Alpine.js?

‚ùå Masalah dengan HTMX Saja:

  • Setiap interaksi butuh server - Toggle dropdown butuh request ke server (waste!)
  • Network latency - UI terasa lambat karena waiting for server response
  • Server load - Server harus handle request untuk hal trivial seperti show/hide modal
  • Tidak bisa offline - Tanpa koneksi, semua interaksi mati total

‚úÖ Alpine.js Solves This:

  • Instant response - Toggle, tabs, dropdown langsung tanpa network delay
  • Zero server load - Pure client-side untuk UI state yang tidak perlu persist
  • Better UX - Feedback langsung, tidak ada loading spinner untuk hal simpel
  • Less bandwidth - Tidak kirim request untuk setiap klik dropdown

üéØ Contoh Real-World:

  • Dropdown menu: HTMX = server request, Alpine.js = instant toggle
  • Modal/Dialog: HTMX = fetch content dari server, Alpine.js = langsung show/hide
  • Tabs: HTMX = load tab content dari server, Alpine.js = switch langsung (jika data sudah ada)
  • Form validation: HTMX = submit dulu baru error, Alpine.js = validate before submit
  • Counter/Timer: HTMX = tidak bisa (butuh server polling), Alpine.js = purely client-side

‚öñÔ∏è Decision Framework:

Use Case HTMX Alpine.js
Submit form, save data ‚úÖ Yes ‚ùå No
Search, load data from DB ‚úÖ Yes ‚ùå No
Toggle dropdown/modal ‚ùå Waste ‚úÖ Yes
Switch tabs (data sudah ada) ‚ùå Waste ‚úÖ Yes
Client-side form validation ‚ùå No ‚úÖ Yes
Dynamic styling/animation ‚ùå No ‚úÖ Yes

üí° Best Practice:

HTMX untuk data persistence (anything yang butuh server/database), Alpine.js untuk UI state (anything yang purely frontend interaction). Keduanya complement each other, bukan kompetisi!

üìö Konsep Pelajaran 6

Konsep Baru: Alpine.js adalah lightweight JavaScript framework untuk menambahkan reactivity dan interactivity langsung di HTML, tanpa perlu round-trip ke server.

üèîÔ∏è Alpine.js Core Directives:

  • x-data - Declare reactive data component
  • x-text - Set text content reactively
  • x-on (@) - Listen to events
  • x-show - Toggle visibility (CSS display)
  • x-if - Conditional rendering (add/remove from DOM)
  • x-model - Two-way data binding
  • x-bind (:) - Bind attributes reactively

‚ö° HTMX vs Alpine.js:

  • HTMX: Server-driven, returns HTML fragments from backend
  • Alpine.js: Client-driven, pure frontend reactivity
  • Best Together: HTMX untuk data persistence, Alpine.js untuk UI interactivity
  • HTMX mengirim request ke server, Alpine.js hanya manipulasi DOM lokal

üéØ Use Cases Alpine.js:

  • Toggle dropdowns, modals, tabs tanpa server request
  • Form validation di client-side sebelum submit
  • Counter, timer, animasi interaktif
  • Dynamic styling berdasarkan state

üéÆ Demo 1: Simple Counter (x-data, x-on, x-text)

üí° Konsep: x-data mendefinisikan reactive state, x-on menangani events, x-text menampilkan data.

Count:

Code:

<div x-data="{ count: 0 }">
    <div x-text="count"></div>
    <button @click="count--">Decrement</button>
    <button @click="count = 0">Reset</button>
    <button @click="count++">Increment</button>
</div>

üéÆ Demo 2: Toggle Visibility (x-show vs x-if)

üí° Konsep: x-show menggunakan CSS display:none, x-if menambah/hapus element dari DOM.

üëã Halo! Ini menggunakan x-show. Element tetap ada di DOM, hanya display:none.

Code:

<div x-data="{ showMessage: false, showAlert: false }">
    <!-- x-show: element tetap di DOM -->
    <button @click="showMessage = !showMessage">Toggle</button>
    <div x-show="showMessage">Message</div>

    <!-- x-if: element ditambah/dihapus dari DOM -->
    <button @click="showAlert = !showAlert">Toggle</button>
    <template x-if="showAlert">
        <div>Alert</div>
    </template>
</div>

üéÆ Demo 4: Dynamic Styling (x-bind:class)

üí° Konsep: x-bind:class (atau :class) untuk menambahkan class secara kondisional. Bisa gunakan object syntax untuk multiple classes.

üîó Combining HTMX + Alpine.js

Best Practice: Gunakan HTMX untuk operasi yang membutuhkan server (CRUD, persistence), dan Alpine.js untuk UI interactivity (dropdown, tabs, modals).

  • HTMX: Form submission, data fetching, search
  • Alpine.js: Toggle UI, client-side validation, animations
  • Together: Form dengan Alpine.js validation + HTMX submission
